# 关于充分利用技术书籍的建议

以下是我在读者询问如何充分利用我的《从头构建大型语言模型》书籍时分享的一些笔记。

我自己阅读技术书籍时也遵循类似的方法。这并非通用的秘诀，但可能是一个有用的起点。

对于这本特定的书，我强烈建议按顺序阅读，因为每一章都依赖于前一章。对于每一章，我建议采取以下步骤。

&nbsp;
### 1) 第一次阅读（离线）

我建议先从头到尾阅读本章，不要编写任何代码。
第一次通读的目标是首先获得大局观。

理想情况下，我建议在远离电脑的地方阅读本章。实体书效果很好，但没有干扰（没有浏览器、社交媒体或电子邮件）的数字设备也可以。

就我个人而言，我在纸上和电子墨水平板电脑上都阅读。虽然我从 2018 年就开始使用电子墨水平板电脑，并且总是尝试更多地在电子墨水上阅读，但我仍然注意到实体书能帮助我更好地集中注意力。这也是为什么我有时会打印具有挑战性或我真正想详细了解的研究论文的原因。

我的建议是让第一次通读成为一个简短、专注的 20 分钟阅读环节，尽量减少干扰，不要过度思考或陷入细节。

标记或注释令人困惑或有趣的部分是可以的，但我不建议在这个阶段查阅资料。我只是建议阅读，但不要运行任何代码。第一遍是为了理解更大的图景。

&nbsp;
### 2) 第二次阅读（带代码）

在第二次通读时，我建议键入并运行本章的代码。复制代码很诱人，因为重新键入是很多工作，但当我阅读其他技术书籍时，它通常有助于我多思考一下代码（相对于只是扫一眼）。

如果我得到的结果与书中不同，我会检查书的 GitHub 仓库并尝试那里的代码。如果我仍然得到不同的结果，我会尝试看看是否是由于不同的包版本、随机种子、CPU/CUDA 等原因造成的。如果我那时仍然无法弄清楚，询问作者也不是一个坏主意（通过书籍论坛、公共 GitHub 仓库问题或讨论，作为最后的手段，电子邮件）。

&nbsp;
### 3) 练习

在第二次通读、重新键入并运行代码之后，通常是尝试练习的好时机。这对于巩固一个人的理解或以半结构化的方式修补问题非常棒。如果练习太具挑战性，看答案是可以的。但是，我仍然建议先给它一个扎实的尝试。

&nbsp;
### 4) 复习笔记并进一步探索

现在，在阅读了这一章、运行了代码并做了练习之后，我建议回到前两次通读的重点和注释，看看是否还有不清楚的地方。

这也是查找其他参考资料或进行快速搜索以澄清任何感觉尚未解决的问题的好时机。但是，即使一切都有意义，阅读更多关于感兴趣的主题也不是一个坏主意。

在这个阶段，将有用的见解、代码片段等写下来或转移到你最喜欢的笔记应用程序也是有意义的。

&nbsp;
### 5) 在项目中使用这些想法

前面的步骤都是关于吸收知识。现在，看看你是否可以在自己的项目中使用某一章的某些方面。或者也许使用书中的代码作为起点构建一个小项目。为了获得灵感，请查看奖励材料，这些基本上是我为了满足自己的好奇心而做的迷你项目。

例如，在阅读了多头注意力机制并实现了 LLM 之后，你可能想知道具有分组查询注意力的模型的表现如何，或者 RMSNorm 与 LayerNorm 真的有多大区别。诸如此类。

也可能有一些较小的方面可以在你自己的项目中有用。例如，有时最终有用的是一个微小的细节，比如测试显式调用 `torch.mps.manual_seed(seed)` 与单独使用 `torch.manual_seed(seed)` 相比是否会改变任何东西。

不过，最终，我还是想以某种方式使用这些知识。这可能涉及使用本章的主要概念，但也涉及我有次学到的一些小知识点，例如，即使是琐碎的事情，比如显式调用 `torch.mps.manual_seed(seed)` 而不仅仅是 `torch.manual_seed(seed)` 是否真的会对我的项目产生影响。

&nbsp;
### 其他想法

当然，以上都不是一成不变的。如果这个主题总体上非常熟悉或简单，而我主要是为了阅读后面的章节而获取一些信息，略读一章是可以的（为了不浪费我的时间）。

此外，对于没有任何代码的章节（例如，介绍性的第 1 章），跳过与代码相关的步骤当然是有意义的。

无论如何，我希望这对你有用。祝你阅读和学习愉快！
